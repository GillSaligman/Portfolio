7-DOF Robotic Arm

Senior Design Project | Drexel University

Overview

Inspired by NASA's RASC-AL competition, my senior design team created a modular robotic arm mounted to a custom chassis with distributed control, built with modularity, adaptability, repairability, and future upgradeability in mind.

Our prototype serves as a hands-on test platform to validate two core concepts:

  • Modular architecture: Streamlined four-wire backbone (HV+/-, LV+/-) drastically reduces wiring complexity
  • Distributed control: Independent joint electronics enable hot-swapping and simplified troubleshooting

Though not intended for actual lunar missions, this system demonstrates essential characteristics for sustainable robotics in challenging environments:

  • Component-level repairability without full system shutdown
  • Future-proof design allowing technology upgrades
  • Redundancy through independent joint controllers

Project Highlights

As team leader, I personally executed all mechanical design, electrical systems, firmware development, prototyping, and validation testing while coordinating documentation and outreach efforts.

Our design process followed NASA's RASC-AL criteria through several phases:

  • Concept Exploration: Evaluated multifunctional robots, specialized swarms, and modular systems
  • Modular Selection: Chose robotic arm approach for long-term serviceability and upgrade potential
  • System Integration: Implemented ROS 2 and MQTT for realistic deployment simulation

Key technical implementations I developed:

  • Distributed control architecture with fail-safe mechanisms
  • Autonomous path planning algorithms
  • End-to-end validation of communication protocols

Mechanical Design

From initial singularity analysis to final implementation, I engineered each joint with precision using torque, payload, and reach calculations to guide iterative prototyping.

The mechanical architecture features:

  • Aluminum backbone: 1"x1" structural tube providing rigidity
  • Material optimization: Strategic use of PLA, PETG, and ABS based on thermal/stress requirements
  • Precision feedback: Potentiometers calibrated via custom regression analysis
  • Motor selection: Pre-emptive torque calculations avoiding iterative dependencies

Key design solutions:

  • Clamping mechanism for ABS components to prevent thermal warping
  • Integrated cable routing through hollow joints
  • Modular mounting system enabling rapid reconfiguration

To overcome torque limitations while minimizing backlash, I engineered a fully 3D-printed harmonic drive featuring:

  • Wave generator: Precision elliptical cam design
  • Flex spline: Flexible yet durable tooth profile
  • Circular spline: Rigid outer gear with 2% more teeth than flex spline

Though time constraints prevented full integration, bench testing demonstrated:

  • 200% torque increase compared to direct drive
  • Near-zero backlash operation
  • Feasibility of high-precision additive manufacturing

Joint 6 presented unique challenges requiring innovative solutions:

  • Rotation requirement: 600° range achieved through 4:1 potentiometer gearing
  • Bearing system: Combined thrust and radial bearings for full constraint
  • Wiring management: Hollow core design enabling internal power to gripper

The parallel-jaw gripper features:

  • Servo-driven mechanism with 120° range of motion
  • Interchangeable jaw system for different payloads
  • Integrated force sensing via current monitoring

Electronics

The electronic architecture implements a shared four-wire backbone (HV+/- and LV+/-) that powers independent joint controllers, enabling modular expansion without system reconfiguration.

Each joint contains a self-contained control system:

  • Microcontroller: ESP8266 selected after rigorous testing of:
    • MicroPython compatibility
    • Analog input resolution
    • Wi-Fi stability under load
  • Sensing: Precision potentiometers for position feedback
  • Actuation: Motor drivers with current-limiting protection

Critical firmware features:

  • Automatic Wi-Fi reconnection routines
  • Hard and soft motor current limits
  • Temperature monitoring with thermal derating

The central control hub implements:

  • Power management:
    • 24V/12V dual-rail power supplies
    • DC-DC buck converters with 92% efficiency
    • Polyfuse protection on all circuits
  • Central processing: Raspberry Pi 4B with dedicated cooling
  • Connectivity: Dual-band Wi-Fi with failover to Ethernet

Design for serviceability:

  • Color-coded wiring harnesses
  • Hot-swappable power modules
  • Diagnostic LEDs on all critical circuits

Software & Communication

Our distributed software architecture uses a Raspberry Pi-hosted network managing MQTT communication across all joints, supporting both position and velocity control modes with real-time telemetry.

The MQTT protocol implements a structured bracket system [DR, DL, J3, J4, J5, J6, J7]:

  • Controller Data:
    • Xbox joystick inputs translated to normalized velocities
    • Deadzone compensation and axis remapping
  • Desired Commands:
    • Position: P[angle] (e.g., P45.00)
    • Velocity: V[value] (e.g., V0.65)
  • Actual Data:
    • Timestamped joint positions
    • Error codes and system status

The central controller implements:

  • Network infrastructure:
    • hostapd for Wi-Fi access point
    • dnsmasq for DHCP services
    • Mosquitto MQTT broker
  • Control modes:
    • Velocity: Direct joystick-to-motor mapping
    • Position: Inverse kinematics calculations
    • Autonomous: Pre-programmed trajectories
  • User interface:
    • SSH command interface
    • Web-based dashboard (Flask)
    • ROS 2 integration for simulation

Each ESP8266 runs autonomous control loops:

  • Startup sequence:
    • Automatic Wi-Fi connection with fallback
    • MQTT topic subscription
    • Self-calibration routine
  • Real-time processing:
    • Bracket command parsing
    • PID control loop at 100Hz
    • Velocity trapezoidal profiling
  • Safety systems:
    • Software limit switches
    • Over-current protection
    • Watchdog timer reset

Prototyping & Assembly

Modular design enabled parallel development with each joint tested and debugged independently before system integration, significantly reducing integration complexity.

Our reverse assembly approach:

  • Began with end-effector (gripper) development
  • Progressed backward through joints (J6 → J5 → ...)
  • Validated each subsystem independently

Key advantages of this methodology:

  • Early validation of most complex components
  • Progressive weight testing as joints were added
  • Cable management perfected at each stage

Software integration strategy:

  • Pre-flashed firmware for all joints
  • Automated PID tuning scripts
  • Continuous integration testing
